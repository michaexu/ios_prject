## 一、产品概述

### 1.1 产品名称
**ReceiptKeeper - AI 收据管家**

### 1.2 产品定位
一款面向全球市场的 AI 驱动收据管理工具，帮助用户通过拍照快速识别收据、自动分类、生成报销单，解决纸质收据管理混乱、报销流程繁琐的痛点。

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| **上班族** | 需要报销差旅/办公费用 | 快速整理收据，生成报销单 |
| **自由职业者** | 需要记账报税 | 自动分类支出，导出税务报告 |
| **小企业主** | 需要管理业务支出 | 多币种支持，批量导出 |

### 1.4 核心价值
- ⚡ **3 秒录入**：拍照 → AI 识别 → 自动分类
- 🌍 **多币种支持**：自动识别币种，实时汇率换算
- 🤖 **AI 智能分类**：准确率 > 90%
- 📄 **一键导出**：PDF 报销单 / CSV 税务报告

### 2.2 功能详细说明

#### 功能 1：拍照识别收据（P0）
**描述：** 用户拍照或上传收据照片，App 使用 OCR + AI 自动提取以下信息：
- 金额（Total）
- 日期（Date）
- 商家名称（Merchant）
- 币种（Currency）
- 税额（Tax，可选）

**验收标准：**
- [ ] 支持拍照和从相册选择
- [ ] OCR 识别准确率 > 85%（清晰照片）
- [ ] 识别速度 < 3 秒
- [ ] 识别失败时有手动输入选项

**技术方案：**
- OCR：Apple Vision Framework（`VNRecognizeTextRequest`）
- AI 分类：本地 Core ML 模型（轻量级），或调用云端 API（GPT-4V / Claude）
- 考虑成本：MVP 阶段优先用本地模型，准确率不够再接入云端

---

#### 功能 2：AI 自动分类（P0）
**描述：** 根据收据内容（商家、金额、时间）自动分类到以下类别：
- 🍽️ 餐饮（Meals）
- ✈️ 交通（Travel）
- 🏨 住宿（Accommodation）
- 💼 办公（Office）
- 📦 其他（Other）

**验收标准：**
- [ ] 分类准确率 > 90%
- [ ] 用户可手动修改分类
- [ ] 支持自定义分类（V1.1）

**技术方案：**
- 本地规则引擎（基于关键词匹配） + Core ML 分类模型
- 关键词示例：餐厅 → 餐饮，Uber → 交通，Hotel → 住宿

---

#### 功能 3：收据列表（P0）
**描述：** 展示所有已识别的收据，支持按时间/分类筛选。

**验收标准：**
- [ ] 列表展示：商家、金额、日期、分类图标
- [ ] 支持按月份筛选
- [ ] 支持按分类筛选
- [ ] 点击进入详情页

**UI 要求：**
- 卡片式布局
- 左滑删除
- 下拉刷新

---

#### 功能 4：手动编辑（P0）
**描述：** 识别结果不准确时，用户可手动修改。

**验收标准：**
- [ ] 所有字段可编辑（金额、日期、商家、分类、备注）
- [ ] 编辑后自动保存
- [ ] 支持删除收据

---

#### 功能 5：导出 PDF 报销单（P0）
**描述：** 用户选择多张收据，生成 PDF 报销单，可直接打印或发送。

**验收标准：**
- [ ] 支持选择多张收据
- [ ] PDF 包含：公司名称（可设置）、收据列表、总金额、日期范围
- [ ] 支持分享到邮件/微信/其他 App
- [ ] PDF 格式美观，适合打印

**技术方案：**
- 使用 `PDFKit` 生成 PDF
- 支持自定义公司名称和logo（V1.1）

## 三、非功能性需求

### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| OCR 识别速度 | < 3 秒 |
| AI 分类速度 | < 1 秒 |
| App 启动速度 | < 2 秒 |
| 列表加载速度 | < 1 秒（100 条数据） |

### 3.2 兼容性
- **iOS 版本**：17.0+（因使用 SwiftData）
- **设备**：iPhone（优先），iPad（适配 V1.1）
- **语言**：英语（MVP），中文/日语/西班牙语（V1.1）

### 3.3 隐私与安全
- 收据数据本地存储（SwiftData）
- iCloud 同步加密
- 不上传原始照片到云端（除非用户授权 AI 优化）
- 符合 GDPR / CCPA 隐私法规

### 3.4 离线能力
- MVP：OCR 和分类需要联网（调用云端 API）
- V1.1：支持离线 OCR（Apple Vision 本地识别）

## 四、技术建议

### 4.1 技术栈
| 模块 | 技术 |
|------|------|
| UI | SwiftUI |
| 数据存储 | SwiftData |
| OCR | Apple Vision Framework |
| AI 分类 | Core ML（本地）或云端 API |
| 云同步 | CloudKit（iCloud） |
| PDF 生成 | PDFKit |
| 多币种 | 固定汇率表（MVP），API（V1.1） |

### 4.2 第三方服务
| 服务 | 用途 | 成本 |
|------|------|------|
| OpenAI GPT-4V API | 收据识别（备选） | $0.01/张 |
| ExchangeRate API | 汇率查询 | 免费（有限额）|
| Google Vision API | OCR（备选） | $1.50/1000 张 |

**建议：** MVP 阶段优先用 Apple Vision（免费）+ 本地规则引擎，准确率达 85% 即可。V1.1 再接入云端 API 提高准确率。

## 七、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| OCR 准确率不高 | 用户体验差 | 优先用 Apple Vision，准确率不够再接入云端 API |
| 云端 API 成本高 | 利润率低 | MVP 阶段限制免费用户调用次数 |
| App Store 审核被拒 | 延迟发布 | 提前阅读审核指南，避免隐私问题 |
| 竞争产品（Expensify） | 用户流失 | 差异化：更简单、更便宜、AI 准确率更高 |

## 八、竞品分析

| App | 优势 | 劣势 | 我们的差异化 |
|-----|------|------|-------------|
| **Expensify** | 功能全面，企业级 | 复杂、贵（$5/月） | 更简单、更便宜 |
| **Smart Receipts** | 自定义分类 | UI 老旧 | 现代 UI、AI 自动分类 |
| **Shoeboxed** | 专业报销 | 需要寄送纸质收据 | 完全数字化 |

**差异化总结：**
- ✅ 更简单（3 步完成：拍照 → 识别 → 导出）
- ✅ 更便宜（免费版可用，Pro $4.99/月）
- ✅ AI 准确率更高（持续学习用户习惯）
