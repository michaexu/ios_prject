# 信息架构（Information Architecture）
## ReceiptKeeper - AI 收据管家

**版本：** V1.0  
**日期：** 2026-07-05  
**作者：** UI 设计师（虚拟员工）

---

## 一、App 结构图

```
ReceiptKeeper
│
├── 📸 拍照识别（Tab 1 - 首页，默认打开相机）
│   ├── 相机页面
│   └── 识别结果页面（可编辑）
│
├── 📋 收据列表（Tab 2）
│   ├── 列表页面（按时间排序）
│   │   ├── 筛选器（按月份/分类）
│   │   └── 搜索框
│   └── 收据详情页面
│
├── 📊 统计（Tab 3）
│   ├── 概览页面（本月支出、分类占比）
│   ├── 饼图（按分类）
│   └── 柱状图（按月）
│
└── ⚙️ 设置（Tab 4）
    ├── 个人信息（公司名称、货币）
    ├── 分类管理
    ├── iCloud 同步状态
    ├── 导出格式设置
    └── 关于/帮助
```

## 二、页面流程图

### 2.1 主流程（拍照识别 → 保存 → 导出）

```
[打开 App]
    ↓
[首页 - 相机]
    ↓ 拍照
[识别中...]
    ↓ OCR + AI 识别（< 3 秒）
[识别结果页面]
    ↓ 用户编辑（可选）
    ↓ 点击"保存"
[收据列表]
    ↓ 用户选择多张收据
    ↓ 点击"导出"
[PDF 预览]
    ↓ 点击"分享"
[系统分享面板] → 邮件/微信/其他 App
```

### 2.2 浏览流程

```
[收据列表]
    ↓ 点击某张收据
[收据详情]
    ↓ 点击"编辑"
[编辑页面]
    ↓ 修改后点击"保存"
[收据详情]（更新后）
```

### 2.3 筛选流程

```
[收据列表]
    ↓ 点击"筛选"按钮
[筛选面板]
    ├── 按月份选择
    └── 按分类选择（多选）
    ↓ 点击"应用"
[收据列表]（显示筛选结果）
```

## 三、导航结构

### 3.1 Tab Bar 导航（4 个 Tab）

| Tab | 图标 | 标题 | 功能 |
|-----|------|------|------|
| 1 | `camera.fill` | Receipts | 拍照识别 + 收据列表 |
| 2 | `chart.pie.fill` | Stats | 统计图表 |
| 3 | `gearshape.fill` | Settings | 设置 |

**注意：** MVP 版本用 3 个 Tab，V1.1 再调整为 4 个。

### 3.2 栈导航（Navigation Stack）

**收据列表 → 收据详情 → 编辑**
```
[收据列表]
    ↓ NavigationLink
[收据详情]
    ↓ NavigationLink
[编辑页面]
    ↓ 保存后 pop
[收据详情]
    ↓ 返回
[收据列表]
```

## 五、用户流程场景

### 场景 1：首次使用（Onboarding）

```
[打开 App]
    ↓
[Splash 页面]（1 秒）
    ↓
[欢迎页面]
    ├── "欢迎使用 ReceiptKeeper"
    ├── "AI 自动识别收据"
    ├── "一键生成报销单"
    └── [开始使用] 按钮
    ↓
[首页 - 相机]
```

**设计要点：**
- Onboarding 只显示一次（用 UserDefaults 记录）
- 3 张滑动卡片，介绍核心功能
- 最后一张卡片请求相机权限

---

### 场景 2：拍照识别收据（核心流程）

```
[首页 - 相机]
    ├── 相机预览（全屏）
    ├── 拍照按钮（底部中间，大圆形按钮）
    └── 相册按钮（左下角）
    ↓ 点击拍照
[识别中...]
    ├── Loading 动画
    └── "正在识别..." 提示
    ↓ 3 秒后
[识别结果页面]
    ├── 收据照片（顶部，可缩放）
    ├── 金额（大字体，可编辑）
    ├── 日期（可编辑）
    ├── 商家（可编辑）
    ├── 分类（可修改）
    ├── 币种（可修改）
    ├── 备注（可输入）
    └── [保存] 按钮
    ↓ 点击保存
[收据列表]（自动滚动到新添加的收据）
```

**设计要点：**
- 识别结果页面：高亮识别字段，让用户快速确认
- 如果识别失败：显示"识别失败，请手动输入"提示

---

### 场景 3：浏览收据列表

```
[收据列表]
    ├── 搜索框（顶部）
    ├── 筛选器（月份选择 + 分类选择）
    └── 收据卡片列表
        ├── 商家名称
        ├── 金额
        ├── 日期
        └── 分类图标
    ↓ 下拉刷新
[列表更新]
    ↓ 上拉加载更多
[加载更多收据]
    ↓ 点击某张收据
[收据详情]
    ├── 收据照片（可点击放大）
    ├── 金额
    ├── 日期
    ├── 商家
    ├── 分类
    ├── 币种
    ├── 备注
    ├── [编辑] 按钮
    └── [删除] 按钮
```

**设计要点：**
- 列表为空时：显示空状态（插画 + "还没有收据，点击拍照添加"）
- 左滑卡片：显示"删除"按钮

---

### 场景 4：导出 PDF 报销单

```
[收据列表]
    ├── [选择] 按钮（右上角）
    ↓ 点击"选择"
[选择模式]
    ├── 每张收据左侧显示复选框
    ├── [全选] 按钮
    └── [导出] 按钮（底部）
    ↓ 选择收据，点击"导出"
[导出设置]
    ├── 公司名称（可编辑）
    ├── 日期范围（自动填充）
    ├── 总金额（自动计算）
    └── [生成 PDF] 按钮
    ↓ 点击"生成 PDF"
[PDF 预览]
    ├── PDF 页面（可缩放）
    └── [分享] 按钮
    ↓ 点击"分享"
[系统分享面板]
    ├── 邮件
    ├── 微信
    ├── 保存到文件
    └── 其他 App
```

**设计要点：**
- PDF 预览：使用 `PDFKit` 渲染
- 分享面板：使用 `UIActivityViewController`

## 六、数据结构

### 6.1 收据数据模型（简化）

```swift
struct Receipt {
    let id: UUID
    let amount: Decimal          // 金额
    let date: Date              // 日期
    let merchant: String        // 商家
    let category: Category      // 分类
    let currency: Currency      // 币种
    let tax: Decimal?           // 税额（可选）
    let note: String?           // 备注（可选）
    let photoURL: URL?          // 照片路径（可选）
    let createdAt: Date         // 创建时间
}
```

### 6.2 分类数据模型

```swift
enum Category: String, CaseIterable {
    case meals = "Meals"           // 餐饮
    case travel = "Travel"         // 交通
    case accommodation = "Accommodation"  // 住宿
    case office = "Office"         // 办公
    case other = "Other"           // 其他
}
```

## 七、页面跳转规则

### 7.1 可以返回的页面
- 识别结果页面 → 首页（取消）
- 编辑页面 → 收据详情（保存/取消）
- 设置页面 → 上一级（返回）

### 7.2 不可以返回的页面
- Splash 页面（自动消失）
- PDF 预览（分享后关闭）
