## Bug 严重程度定义

### P0 - 致命（Blocker）
- 应用崩溃、闪退
- 核心功能完全不可用（无法开始训练）
- 数据丢失
- 安全问题

**处理要求**：必须立即修复，阻塞发布

### P1 - 严重（Critical）
- 重要功能不可用（声音/震动不工作）
- 影响用户体验（倒计时错误）
- 性能问题（卡顿、超时）

**处理要求**：必须修复，否则不能发布

### P2 - 一般（Major）
- 次要功能问题（统计数据显示错误）
- UI 问题（文字截断、颜色错误）
- 兼容性问题（某个机型显示异常）

**处理要求**：尽量修复，如无法修复可推迟到下个版本

### P3 - 轻微（Minor）
- 界面细节问题（间距不对、字体大小不合适）
- 优化建议
- 边缘情况

**处理要求**：可修复可不修复，根据时间决定

## Bug 状态流转

```
新建 (New)
  ↓
处理中 (In Progress)
  ↓
已修复 (Fixed) ← 【开发人员修复后】
  ↓
验证中 (Verifying) ← 【测试人员验证】
  ↓
已关闭 (Closed) ← 【验证通过】
  ↓
重新打开 (Reopened) ← 【验证失败，返回处理中】
```

## Bug 报告示例

### 示例 1：倒计时精度问题

**Bug ID**：BUG-001  
**标题**：计时器倒计时误差超过 1 秒  
**严重性**：P1  
**优先级**：高  

**问题描述**：  
计时器运行 5 分钟后，倒计时显示的时间比实际时间快 2 秒。

**复现步骤**：
1. 打开 App
2. 开始训练（任意方案）
3. 运行 5 分钟
4. 对比系统时钟

**预期结果**：误差 < 0.1 秒  
**实际结果**：误差 2 秒  

**根因分析**：使用 `Timer.scheduledTimer` 精度不够  
**修复方案**：改用 `DispatchSourceTimer`  

**验证结果**：通过（误差 < 0.05 秒）
