# PRD - Interval Timer（间歇性训练计时器）

## 1. 产品概述

### 1.1 产品名称
**Interval Timer - 简易间歇训练计时器**

### 1.2 产品定位
一款专为健身房训练场景设计的间歇性训练计时器，让用户能够快速设计训练方案并高效完成训练。

### 1.3 目标用户
- **主要用户**：健身爱好者（涵盖新手到达人）
- **使用场景**：健身房自由重量训练、器械训练
- **用户痛点**：
  - 现有 App 操作复杂，设置耗时
  - 无法快速开始训练
  - 训练计划切换不方便

### 1.4 核心价值
**"3 秒开始训练"** - 最简单的间歇训练计时器

---

## 2. 用户故事（User Stories）

### 2.1 核心用户故事
1. **作为用户，我希望快速开始训练**，不需要复杂设置，点击即可开始
2. **作为用户，我希望自定义训练和休息时间**，适应不同训练需求
3. **作为用户，我希望有预设训练计划**，可以直接使用经典方案（如 Tabata）
4. **作为用户，我希望训练时有声音和震动提醒**，不需要一直看屏幕
5. **作为用户，我希望查看训练记录**，了解自己的训练频率和时长

### 2.2 次要用户故事
6. **作为用户，我希望保存自定义训练方案**，下次快速调用
7. **作为用户，我希望训练时屏幕常亮**，方便随时查看
8. **作为用户，我希望调整提示音**，适应不同环境（安静/嘈杂）

## 3. MVP 功能清单（V1.0）

### 3.1 功能 1：基础计时器 ✅
**描述**：自定义工作/休息时间间隔

**详细需求**：
- 设置训练时间（1秒 - 99分钟）
- 设置休息时间（1秒 - 99分钟）
- 设置循环次数（1 - 99次）
- 大字体显示倒计时
- 训练和休息状态用不同颜色区分（训练=红色，休息=绿色）

**界面元素**：
- 数字输入框（训练时间、休息时间、循环次数）
- 开始/暂停按钮
- 重置按钮
- 当前状态显示（训练中/休息中/准备中）

### 3.2 功能 2：预设训练计划 ✅
**描述**：提供经典训练方案，一键使用

**预设方案**：
1. **Tabata**（20秒训练 + 10秒休息，8轮）
2. **7分钟训练**（30秒训练 + 10秒休息，12轮）
3. **HIIT 初级**（45秒训练 + 15秒休息，10轮）
4. **HIIT 高级**（60秒训练 + 20秒休息，15轮）
5. **自定义**（用户自己设置）

**交互**：
- 首页显示预设方案卡片
- 点击卡片，自动填充计时器参数
- 可以编辑预设方案的参数

### 3.3 功能 3：声音/震动提醒 ✅
**描述**：训练状态变化时提醒用户

**提醒时机**：
- 训练开始（"开始训练"语音 + 震动）
- 训练结束（提示音 + 震动）
- 休息结束（"准备训练"语音 + 震动）
- 全部训练完成（完成音效 + 长震动）

**设置选项**：
- 开启/关闭声音
- 开启/关闭震动
- 选择提示音（3种可选）

### 3.4 功能 4：训练记录统计 ✅
**描述**：记录用户的训练历史

**记录数据**：
- 训练日期
- 训练方案名称
- 总训练时长
- 完成的循环数

**统计展示**：
- 本周训练次数
- 本周总训练时长
- 连续训练天数
- 训练历史列表（按日期排序）

## 4. 非功能性需求

### 4.1 性能
- 计时精度：误差 < 0.1秒
- 启动速度：冷启动 < 2秒
- 响应速度：点击按钮响应 < 0.1秒

### 4.2 兼容性
- **最低系统**：iOS 15.0
- **设备**：iPhone（暂不支持 iPad）
- **横竖屏**：仅支持竖屏

### 4.3 离线使用
- 所有功能支持离线使用
- 无需注册登录

## 5. 技术建议（供开发参考）

### 5.1 技术栈
- **语言**：Swift 5
- **UI 框架**：SwiftUI（推荐）/ UIKit
- **数据存储**：UserDefaults（设置）+ CoreData（训练记录）
- **音频**：AVFoundation
- **震动**：CoreHaptics

### 5.2 关键实现点
- 使用 `Timer` 或 `DispatchSourceTimer` 实现高精度计时
- 后台运行支持（如果需要，可申请后台权限）
- 屏幕常亮：`UIApplication.shared.isIdleTimerDisabled = true`

## 8. 风险与假设

### 8.1 风险
- **竞争激烈**：间歇训练计时器品类已有多个成熟产品
  - **应对**：强调"简单"和"快速"，做差异化
  
- **用户留存**：工具类 App 容易用完即走
  - **应对**：通过训练记录统计，增加用户粘性

### 8.2 假设
- 用户希望快速开始训练，不愿意花时间设置
- 健身房场景用户更喜欢简单直观的界面
- 训练记录功能可以增加用户留存
